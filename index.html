  • Co-creating Barebones, a templating language/engine for the JVM (based off of JSP, if you're familiar).
  • Co-creating WhatVersionOfGosuAmIUsing, a web-based file browser/searcher that allows Guidewire engineers to more easily traverse through application release files in order to make the task of identifying the Gosu version that particular Guidewire product/release uses.

Projects

Note: I am working on creating more detailed write-ups/reflections for my projects; for now, links will lead to the Github repo for the particular project.

  • Single-Person DBMS (Fall 2017): Includes implementations of B+ tree, Table Iterators, Nested Loops Join, External Sort, Sort-Merge Join, Query Optimizer. Done in class (CS 186).
  • LunchMeet (Summer 2017): Web application that maintains a userbase and matches users with each other for lunch based on availabilities, shared interests, food restrictions, etc. Built at LinkedIn Hackathon.
  • Barebones (Summer 2017): A templating engine for the JVM that's removed from the servlet infrastructure. Additionally, it supports typed arguments to templates in order to take advantage of Java's static type-checking. We built a lexer and a program that turns Barebones files into valid Java files. Built at Guidewire.
  • WhatVersionOfGosuAmIUsing (Summer 2017): An internal tool for Guidewire - essentially, it's a web-based file browser/searcher that helps Guidewire engineers identify the version of Gosu that a particular release of a particular product is using. The issue that the tool helps alleviate is the nesting of .jar and .zip files within each other - there's no easy way to easily unpack .jar files within .jar files without unpacking the entire outer file, and traversing large and complicated build files using a terminal is often difficult and tedious. We provide a natural way to traverse/search through these files. Notably, we used Barebones as the front-end "view" of the web application (pretty cool, if you ask me!). Built at Guidewire.
